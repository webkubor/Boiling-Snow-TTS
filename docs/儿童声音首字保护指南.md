# 🧒 儿童声音首字保护指南

## 问题现象
儿童声音生成时，经常出现**首字被截断**或**开头卡一下**的感觉，像是音频从中间开始播放。

## 原因分析
1. **TTS 模型特性**：儿童声音通常音调较高、语速快，首字气音轻，容易被识别为静音而截断
2. **参考音频裁剪**：自动裁剪静音时过度处理了开头的轻音
3. **生成参数**：未对首字进行特殊保护

---

## ✅ 解决方案

### 方案一：Tone 指令中加入首字保护（推荐）

```json
{
  "tone": "七八岁女童，声音软糯甜美。...首字清晰入字，开头绝对禁止截断或吞音，第一句完整发出"
}
```

**关键词**：
- `首字清晰入字`
- `开头绝对禁止截断`
- `第一句完整发出`
- `禁止吞音`

---

### 方案二：Text 开头加空格或语气词

```json
{
  "text": " 请问这位少侠..."  // 开头加空格
}
```

或

```json
{
  "text": "嗯~请问这位少侠..."  // 开头加语气词作为缓冲
}
```

---

### 方案三：调整参考音频提取参数

在克隆模式下，减少 `skip_start_ms` 的值：

```python
# core/modes/cloner.py 中
extract_voice_seed(ref_path, persona_cn, max_sec=10, skip_start_ms=500)  # 从 1500 改为 500
```

---

### 方案四：后期手动修复

如果已经生成，可以用音频编辑软件：
1. 在开头添加 100-200ms 的空白
2. 或用淡入效果让首字自然进入

## 📋 儿童声音设计模板

### 天真型
```json
{
  "text": " 师父，江湖是什么呀？",
  "tone": "七八岁男童，声音稚嫩清亮，语速轻快自然，尾音微微上扬。首字清晰入字，禁止截断",
  "emotion": "天真，好奇"
}
```

### 机灵型
```json
{
  "text": " 嘿嘿，我早学会了！",
  "tone": "八九岁男童，声音清脆灵动，语速偏快，咬字利落。开头禁止吞音，首字完整",
  "emotion": "得意，调皮"
}
```

### 温婉型
```json
{
  "text": " 请问少侠，前方可有客栈？",
  "tone": "七八岁女童，声音软糯甜美，语速轻柔。首字清晰，开头不截断",
  "emotion": "温婉，礼貌"
}
```

### 怯懦型
```json
{
  "text": " 我、我一个人害怕...",
  "tone": "六七岁孩童，声音细弱软糯，语速慢。首字虽轻但完整发出，不截断",
  "emotion": "胆怯，委屈"
}
```

---

## 🔧 生产建议

1. **设计阶段**：先在 text 前加空格测试效果
2. **确认满意后**：落库时可以去掉空格，但保留 tone 中的首字保护指令
3. **批量生成**：用同一套防截断模板，确保一致性
